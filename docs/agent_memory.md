# Архитектура Agent Memory

Документация описывает систему памяти агентов в HordeForge, которая позволяет агентам использовать историю предыдущих решений и патчей для более эффективной работы.

## Обзор

Agent Memory - это система хранения и извлечения информации о предыдущих задачах, патчах и архитектурных решениях. Она позволяет агентам использовать опыт решения аналогичных задач в прошлом, что повышает эффективность и качество генерации кода.

## Архитектура

### Компоненты системы

1. **Memory Collections** - коллекции в векторной базе Qdrant для хранения различных типов памяти:
   - `memory_tasks` - информация о выполненных задачах
   - `memory_patches` - патчи и изменения кода
   - `memory_decisions` - архитектурные решения

2. **Memory Store** - интерфейс для записи и чтения памяти из Qdrant

3. **Memory Hook** - компонент, автоматически записывающий память после успешных шагов pipeline

4. **Memory Retriever** - компонент для поиска релевантных записей памяти

5. **Context Builder** - объединяет память с RAG-контекстом репозитория

## Типы записей памяти

Система поддерживает три типа записей:

### Task Entry
Хранит информацию о выполненных задачах:
- task_description: описание задачи
- pipeline: название пайплайна
- agents_used: использованные агенты
- result_status: статус результата

### Patch Entry
Хранит информацию о внесенных изменениях:
- task_description: описание задачи
- file: имя файла
- diff: текст изменений
- reason: причина изменений

### Decision Entry
Хранит архитектурные решения:
- task_description: описание задачи
- architecture_decision: принятое решение
- context: контекст принятия решения
- result: результат/последствия

## Автоматическая запись памяти

После успешного выполнения шагов pipeline автоматически вызывается Memory Hook, который анализирует результат и создает соответствующую запись в памяти. Запись происходит только для успешных шагов (со статусом SUCCESS, MERGED и т.д.).

### Интеграция с Orchestrator Engine

Memory Hook интегрирован в Orchestrator Engine и вызывается после каждого успешного шага. Это обеспечивает автоматическое заполнение памяти информацией о выполненных задачах.

## Использование памяти агентами

Агенты могут использовать память следующими способами:

1. **Поиск аналогичных задач** - поиск предыдущих решений схожих задач
2. **Использование успешных паттернов** - применение проверенных решений
3. **Избежание прошлых ошибок** - предотвращение повторения ошибок

## Контекст сборки

Context Builder объединяет:
- Контекст из памяти (Previous solutions)
- Контекст из RAG репозитория (Repository context)

Это позволяет агентам использовать как исторические данные, так и актуальную информацию из репозитория.

## Безопасность и ограничения

- Запись в память происходит только после успешного выполнения шагов
- Система обрабатывает ошибки записи без прерывания выполнения pipeline
- Поддерживается версионирование схемы записей
