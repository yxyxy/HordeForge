# Задача P17: Оптимизация RAG-компонента по образцу Local RAG

## Описание
Необходимо оптимизировать текущий RAG-компонент в соответствии с архитектурой Local RAG для достижения более высокой скорости индексации (менее 1 минуты вместо текущего часа).

## Цели
1. Реализовать структурированный подход к индексации: парсинг → извлечение символов → разбиение на чанки → эмбеддинги → сохранение
2. Добавить пакетную обработку для ускорения вычисления эмбеддингов
3. Оптимизировать разбиение на чанки с учетом структуры кода (функции, классы)

## Технические требования

### 1. Структурированный подход к индексации
- Реализовать четко разделенные этапы обработки
- Использовать Tree-sitter для точного парсинга кода
- Извлекать символы (функции, классы, методы) с метаданными
- Создавать чанки на основе структуры кода, а не простого деления по строкам

### 2. Пакетная обработка
- Реализовать пакетное вычисление эмбеддингов для ускорения процесса
- Обеспечить нормализацию векторов в пакетном режиме
- Оптимизировать сохранение в векторную базу данных

### 3. Оптимизация разбиения на чанки
- Использовать информацию о структуре кода при разбиении
- Создавать чанки на основе функций, классов и других значимых элементов
- Обеспечить перекрытие чанков для сохранения контекста

## Подзадачи

### P17.1: Интеграция Tree-sitter парсеров
- [ ] Добавить зависимости для Tree-sitter (tree-sitter, tree-sitter-python, tree-sitter-javascript и др.)
- [ ] Создать модуль для парсинга файлов с использованием Tree-sitter
- [ ] Реализовать извлечение символов (функций, классов, методов) с помощью Tree-sitter запросов
- [ ] Обновить `SymbolExtractor` для использования Tree-sitter

### P17.2: Оптимизация процесса индексации
- [ ] Разделить процесс индексации на четкие этапы: парсинг → извлечение символов → разбиение на чанки → эмбеддинги → сохранение
- [ ] Создать отдельные классы/функции для каждого этапа
- [ ] Обеспечить передачу данных между этапами с сохранением метаданных
- [ ] Добавить логирование прогресса для каждого этапа

### P17.3: Пакетная обработка эмбеддингов
- [ ] Модифицировать `IngestionPipeline` для поддержки пакетной обработки
- [ ] Реализовать буферизацию текстов для вычисления эмбеддингов пакетами
- [ ] Оптимизировать размеры батчей для максимальной производительности
- [ ] Обеспечить нормализацию векторов в пакетном режиме

### P17.4: Умное разбиение на чанки
- [ ] Разработать алгоритм разбиения на чанки с учетом структуры кода
- [ ] Использовать информацию о символах для определения границ чанков
- [ ] Обеспечить перекрытие чанков для сохранения контекста
- [ ] Добавить настройки для контроля размера чанков и перекрытия

### P17.5: Интеграция и тестирование
- [ ] Объединить все компоненты в обновленный `RagInitializer`
- [ ] Провести тестирование производительности
- [ ] Сравнить время индексации с текущей реализацией
- [ ] Обеспечить обратную совместимость с существующими компонентами

## Ожидаемые результаты
- Время индексации уменьшено до менее чем 1 минуты для типичного репозитория
- Улучшенная точность извлечения релевантных фрагментов кода
- Более структурированное хранение информации о символах
- Повышенная общая производительность RAG-компонента

## Метрики успеха
- Время индексации уменьшено минимум в 10 раз
- Количество правильно извлеченных символов увеличено
- Качество поиска не ухудшено
- Обратная совместимость сохранена
