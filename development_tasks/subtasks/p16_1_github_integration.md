# Подзадача P16.1: Реализовать GitHub интеграцию

## Описание
Реализовать полноценную интеграцию с GitHub API для работы с issue, PR, комментариями и другими элементами репозитория.

## Приоритет
HIGH

## Технические детали

### 1. Агенты для реализации
- `agents/github_client.py` - основной клиент для взаимодействия с GitHub API
- Обновить `agents/dod_extractor.py` для работы с реальными GitHub issue
- Обновить `agents/review_agent.py` для работы с реальными PR
- Обновить `agents/pr_merge_agent.py` для выполнения реальных действий с PR

### 2. Функциональность
- Чтение GitHub issue и извлечение требований
- Создание и обновление комментариев в PR
- Слияние PR после успешного ревью
- Обработка GitHub вебхуков
- Работа с labels, assignees, проектами

### 3. Интеграции
- GitHub REST API v3
- GitHub GraphQL API (для сложных запросов)
- GitHub Apps или Personal Access Token аутентификация

## Критерии готовности (DoD)
- [ ] Реализован клиент для GitHub API
- [ ] Агенты могут читать реальные GitHub issue
- [ ] Агенты могут создавать комментарии в PR
- [ ] Агенты могут сливать PR при положительном ревью
- [ ] Написаны тесты для GitHub интеграции
- [ ] Обновлена документация

## Риски
- Ограничения по rate limiting от GitHub API
- Проблемы с аутентификацией и авторизацией
- Ошибки при работе с различными типами репозиториев

## Комментарии
Необходимо предусмотреть обработку ошибок и повторные попытки при работе с GitHub API. Также важно реализовать кэширование для часто используемых данных.
