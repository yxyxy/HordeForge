# Подзадача P16.2: Реализовать LLM интеграцию

## Описание
Реализовать полноценную интеграцию с LLM (Large Language Model) для генерации кода, анализа, ревью и других AI-функций.

## Приоритет
HIGH

## Технические детали

### 1. Агенты для реализации
- `agents/llm_wrapper.py` - универсальный интерфейс для различных LLM
- Обновить `agents/code_generator.py` для генерации кода с использованием LLM
- Обновить `agents/specification_writer.py` для генерации спецификаций
- Обновить `agents/test_generator.py` для генерации тестов
- Обновить `agents/review_agent.py` для автоматического ревью кода

### 2. Функциональность
- Поддержка различных LLM провайдеров (OpenAI, Anthropic, Ollama)
- Управление контекстом и токенами
- Температура и параметры генерации
- Обработка ошибок и повторные попытки
- Логирование и трассировка вызовов LLM

### 3. Интеграции
- OpenAI GPT API
- Anthropic Claude API
- Локальные модели через Ollama
- Поддержка различных форматов (text, JSON, code)

## Критерии готовности (DoD)
- [ ] Реализован универсальный LLM клиент
- [ ] Поддерживаются несколько провайдеров LLM
- [ ] Агенты могут генерировать код с помощью LLM
- [ ] Агенты могут генерировать тесты с помощью LLM
- [ ] Агенты могут анализировать и ревьювить код с помощью LLM
- [ ] Написаны тесты для LLM интеграции
- [ ] Обновлена документация

## Риски
- Высокая стоимость использования внешних LLM API
- Ограничения по длине контекста
- Качество генерации может быть недостаточным для сложных задач

## Комментарии
Необходимо предусмотреть возможность переключения между различными LLM провайдерами и настройки параметров генерации в зависимости от типа задачи.
