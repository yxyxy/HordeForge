# Подзадача P17.4: Умное разбиение на чанки

## Описание
Необходимо разработать алгоритм разбиения на чанки с учетом структуры кода, используя информацию о символах для определения границ чанков.

## Цели
1. Разработать алгоритм разбиения на чанки с учетом структуры кода
2. Использовать информацию о символах для определения границ чанков
3. Обеспечить перекрытие чанков для сохранения контекста
4. Добавить настройки для контроля размера чанков и перекрытия

## Технические требования

### 1. Структурное разбиение
- Использовать информацию о символах (функции, классы, методы) для определения границ чанков
- Создавать чанки на основе логических единиц кода, а не фиксированного количества строк
- Обеспечить целостность логических блоков кода внутри чанков

### 2. Перекрытие чанков
- Реализовать перекрытие чанков для сохранения контекста
- Обеспечить настраиваемый размер перекрытия
- Поддерживать перекрытие на уровне логических блоков (например, включать связанные функции)

### 3. Настройки разбиения
- Обеспечить возможность настройки максимального размера чанка
- Поддерживать настройку размера перекрытия
- Реализовать настройку минимального размера чанка
- Обеспечить гибкость алгоритма для различных языков программирования

## Подробное описание

### 1. Анализатор структуры кода (CodeStructureAnalyzer)
Класс `CodeStructureAnalyzer`:
- Принимает исходный код файла и извлеченные символы
- Определяет иерархию структурных элементов (классы, функции, методы)
- Создает карту расположения элементов в коде
- Определяет связи между элементами (например, методы внутри классов)

### 2. Алгоритм разбиения (SmartChunker)
Класс `SmartChunker`:
- Использует информацию от `CodeStructureAnalyzer` для разбиения
- Создает чанки на основе логических блоков кода
- Обеспечивает целостность функций, классов и других структурных элементов
- Поддерживает настраиваемые параметры размера и перекрытия

### 3. Генератор чанков (ChunkGenerator)
Класс `ChunkGenerator`:
- Принимает структурированное представление кода
- Генерирует чанки с учетом структуры и настроек
- Обеспечивает перекрытие чанков при необходимости
- Возвращает чанки с полной информацией о границах и связаных символах

### 4. Интеграция с существующим кодом
- Обновить `ChunkingStage` для использования умного разбиения
- Обеспечить совместимость с существующим интерфейсом
- Поддерживать опциональное использование старого алгоритма для сравнения
