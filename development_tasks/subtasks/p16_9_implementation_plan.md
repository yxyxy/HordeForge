# Подзадача P16.9: План реализации LLM wrapper с CLI интерфейсом

## Описание
Создать детальный план реализации всех компонентов LLM wrapper с CLI интерфейсом, включая последовательность разработки и зависимости

## Приоритет
HIGH

## Технические детали

### 1. Подготовительные работы
- [ ] Анализ архитектуры Cline для понимания структуры провайдеров
- [ ] Подготовка архитектурных изменений в существующем `llm_wrapper.py`
- [ ] Создание интерфейса ApiHandler совместимого с Cline
- [ ] Подготовка системы типов для обработки сообщений и стриминга

### 2. Этап 1: Базовая инфраструктура (Week 1)
- [ ] Реализовать базовый интерфейс ApiHandler
- [ ] Добавить систему retry с обратными вызовами
- [ ] Реализовать базовую поддержку стриминга
- [ ] Добавить поддержку инструментов (tools)
- [ ] Создать систему обработки сообщений (ApiStream)

### 3. Этап 2: Интеграция провайдеров (Week 2-3)
- [ ] Интегрировать Ollama провайдер из Cline
- [ ] Интегрировать Gemini провайдер из Cline
- [ ] Интегрировать OpenRouter провайдер из Cline
- [ ] Интегрировать AWS Bedrock провайдер из Cline
- [ ] Интегрировать Vertex AI провайдер из Cline
- [ ] Интегрировать остальные провайдеры (LM Studio, DeepSeek, Fireworks, и т.д.)

### 4. Этап 3: Система токен-бюджета (Week 3)
- [ ] Реализовать ModelInfo с поддержкой цен и ограничений
- [ ] Добавить поддержку tiered pricing
- [ ] Реализовать систему reasoning budget
- [ ] Добавить поддержку prompt cache (cacheWritesPrice, cacheReadsPrice)
- [ ] Интегрировать систему мониторинга с каждым провайдером

### 5. Этап 4: CLI интерфейс (Week 4)
- [ ] Реализовать команду `cline` для интерактивной работы
- [ ] Добавить команды `--provider` и `--model`
- [ ] Реализовать режимы `--plan` и `--act`
- [ ] Добавить команду `--settings` для настройки провайдеров
- [ ] Реализовать интерактивный чат с историей сообщений
- [ ] Добавить команды для просмотра токен-использования и стоимости

### 6. Этап 5: Интеграция с HordeForge (Week 4-5)
- [ ] Интегрировать с существующим `cli.py`
- [ ] Обеспечить совместимость с существующими агентами
- [ ] Добавить поддержку новых провайдеров в агенты
- [ ] Обновить конфигурацию для новых провайдеров

### 7. Этап 6: Тестирование и совместимость (Week 5-6)
- [ ] Написать тесты для каждого нового провайдера
- [ ] Проверить стриминг для каждого провайдера
- [ ] Проверить поддержку инструментов
- [ ] Проверить обработку ошибок
- [ ] Запустить все существующие тесты с новой системой
- [ ] Проверить работу пайплайнов с новым wrapper

### 8. Этап 7: Документация и финализация (Week 6)
- [ ] Обновить документацию для новых функций
- [ ] Создать руководства по использованию CLI
- [ ] Добавить примеры конфигурации провайдеров
- [ ] Провести финальное тестирование

## Зависимости
- Зависит от: P16.1 (GitHub интеграция), P16.2 (LLM интеграция)
- Блокирует: P16.3 (Feature pipeline agents)

## Критерии готовности (DoD)
- [ ] Все провайдеры из Cline интегрированы
- [ ] CLI интерфейс полностью функционален
- [ ] Система токен-бюджета работает корректно
- [ ] Все тесты проходят
- [ ] Обратная совместимость обеспечена
- [ ] Документация обновлена

## Риски
- Сложность интеграции большого количества провайдеров
- Возможные конфликты с существующей архитектурой
- Проблемы с производительностью при большом количестве провайдеров

## Комментарии
План предусматривает поэтапную разработку с четкими Milestone для контроля прогресса и раннего выявления проблем.
