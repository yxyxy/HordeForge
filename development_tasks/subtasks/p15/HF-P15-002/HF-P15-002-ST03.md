### Scenario

HF-P15-002-ST03 Агент использует memory context при генерации кода

Given в memory есть похожие задачи из прошлого
When code_generator получает задачу "fix auth validation"
Then промпт содержит:
```
Previous solutions:
- Task: Fix login validation (2024-01-15)
  Diff: Added null check in validate_user()
  Result: MERGED
- Task: Add token refresh (2024-01-10)
  ...
```
And агент может использовать паттерны из прошлых решений

## Тест-дизайн

```python
def test_context_builder_includes_memory():
    # Arrange
    memory_results = [
        {"task_description": "Fix auth bug", "diff": "...", "result_status": "MERGED"}
    ]
    repo_results = [{"content": "def validate():", "file": "auth.py"}]
    
    # Act
    context = build_agent_context(
        query="fix authentication",
        memory_results=memory_results,
        repo_results=repo_results
    )
    
    # Assert
    assert "Previous solutions:" in context
    assert "Fix auth bug" in context
    assert "Repository context:" in context
    assert "def validate():" in context
    
def test_context_builder_respects_token_limit():
    # Arrange
    large_memory = [{"task_description": "x" * 1000} for _ in range(10)]
    
    # Act
    context = build_agent_context("query", large_memory, [], max_tokens=500)
    
    # Assert
    assert len(context) <= 600  # with buffer
```
