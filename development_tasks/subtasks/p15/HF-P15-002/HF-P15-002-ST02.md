## BDD постановка

### Feature

HF-P15-002 Agent Memory System

### Scenario

HF-P15-002-ST02 Автоматическая запись memory после успешного review

Given pipeline выполняет шаг review_agent
When шаг завершается со статусом SUCCESS
Then автоматически вызывается memory_writer
And создается PatchEntry с:
- task_description из issue
- diff из сгенерированного кода
- agents_used: [planners, code_generator, review]
- result_status: SUCCESS/NEEDS_FIX/MERGED

## Тест-дизайн

```python
def test_memory_hook_on_success():
    # Arrange
    hook = MemoryHook(memory_store=mock_store)
    step_result = {
        "status": "SUCCESS",
        "artifacts": [{"type": "patch", "content": {"diff": "..."}}]
    }
    
    # Act
    hook.after_step("code_generator", step_result, context)
    
    # Assert
    mock_store.add_memory.assert_called_once()
    
def test_memory_hook_on_failure():
    # Arrange
    hook = MemoryHook(memory_store=mock_store)
    step_result = {"status": "FAILED", "artifacts": []}
    
    # Act
    hook.after_step("code_generator", step_result, context)
    
    # Assert
    mock_store.add_memory.assert_not_called()
```
