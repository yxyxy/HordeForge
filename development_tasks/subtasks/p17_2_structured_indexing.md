# Подзадача P17.2: Оптимизация процесса индексации

## Описание
Необходимо разделить процесс индексации на четкие этапы: парсинг → извлечение символов → разбиение на чанки → эмбеддинги → сохранение, с возможностью отслеживания прогресса на каждом этапе.

## Цели
1. Разделить процесс индексации на четко определенные этапы
2. Создать отдельные классы/функции для каждого этапа
3. Обеспечить передачу данных между этапами с сохранением метаданных
4. Добавить логирование прогресса для каждого этапа

## Технические требования

### 1. Архитектура этапов
- Каждый этап должен быть изолирован в отдельный класс или функцию
- Должна быть возможность запускать каждый этап отдельно или в составе общего процесса
- Данные между этапами передаются в стандартизированном формате

### 2. Сохранение метаданных
- Все метаданные (файл, строка начала/конца, тип символа и т.д.) должны передаваться между этапами
- Должна быть поддержка расширения метаданных на каждом этапе
- Метаданные должны быть доступны при поиске

### 3. Логирование прогресса
- Для каждого этапа должно быть детальное логирование прогресса
- Должна быть возможность отслеживания времени выполнения каждого этапа
- Логи должны содержать статистику по обработанным элементам

## Подробное описание

### 1. Этап парсинга (ParsingStage)
Класс `ParsingStage`:
- Принимает список файлов для обработки
- Использует TreeSitterParser для парсинга файлов
- Возвращает список объектов `ParsedFile` с AST и информацией о файле
- Логирует количество обработанных файлов и время выполнения

### 2. Этап извлечения символов (SymbolExtractionStage)
Класс `SymbolExtractionStage`:
- Принимает список `ParsedFile`
- Использует TreeSitterSymbolExtractor для извлечения символов
- Возвращает список объектов `Symbol` с информацией о типе, имени, расположении
- Логирует количество извлеченных символов и время выполнения

### 3. Этап разбиения на чанки (ChunkingStage)
Класс `ChunkingStage`:
- Принимает список `Symbol` и исходный код файлов
- Создает чанки на основе структуры кода (функции, классы)
- Возвращает список объектов `Chunk` с текстом, метаданными и связями с символами
- Логирует количество созданных чанков и время выполнения

### 4. Этап вычисления эмбеддингов (EmbeddingStage)
Класс `EmbeddingStage`:
- Принимает список `Chunk`
- Вычисляет эмбеддинги для текста чанков с использованием пакетной обработки
- Возвращает список `Chunk` с добавленными векторами эмбеддингов
- Логирует время вычисления и количество обработанных чанков

### 5. Этап сохранения (StorageStage)
Класс `StorageStage`:
- Принимает список `Chunk` с эмбеддингами
- Сохраняет в векторное хранилище (Qdrant) с сохранением метаданных
- Логирует количество сохраненных элементов и время выполнения

### 6. Оркестратор индексации (IndexingOrchestrator)
Класс `IndexingOrchestrator`:
- Управляет последовательным выполнением всех этапов
- Обрабатывает ошибки и обеспечивает восстановление
- Предоставляет общий интерфейс для индексации
- Логирует прогресс выполнения по всем этапам
