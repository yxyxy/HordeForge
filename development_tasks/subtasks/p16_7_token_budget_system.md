# Подзадача P16.7: Система токен-бюджета и мониторинга стоимости

## Описание
Внедрить систему токен-бюджета и мониторинга стоимости с полной совместимостью с Cline

## Приоритет
HIGH

## Технические детали

### 1. ModelInfo и цены
Источник: https://github.com/cline/cline/blob/main/src/shared/api/types.ts

- [ ] Реализовать ModelInfo с поддержкой цен и ограничений как в Cline
- [ ] Добавить поддержку tiered pricing для моделей
- [ ] Реализовать систему reasoning budget
- [ ] Добавить поддержку prompt cache (cacheWritesPrice, cacheReadsPrice)

### 2. Система мониторинга
- [ ] Добавить отслеживание input/output токенов
- [ ] Реализовать подсчет стоимости по моделям
- [ ] Добавить поддержку cache token accounting
- [ ] Реализовать систему бюджетного контроля

### 3. Интеграция с провайдерами
- [ ] Обновить каждый провайдер для отправки токен-информации
- [ ] Добавить обработку токен-использования из API ответов
- [ ] Реализовать расчет стоимости для каждого провайдера
- [ ] Добавить поддержку reasoning token учета

### 4. CLI команды для мониторинга
- [ ] Добавить команду `--tokens` для просмотра использования
- [ ] Реализовать команду `--cost` для просмотра стоимости
- [ ] Добавить команду `--budget` для просмотра бюджета
- [ ] Реализовать команду `--usage-history` для истории

## Критерии готовности (DoD)
- [ ] ModelInfo полностью совместим с Cline
- [ ] Система мониторинга стоимости работает
- [ ] Каждый провайдер правильно отслеживает токены
- [ ] CLI команды для мониторинга работают
- [ ] Тесты покрывают систему бюджета
- [ ] Документация обновлена

## Риски
- Сложность точного расчета стоимости для разных провайдеров
- Различия в форматах токен-информации между API
- Совместимость с существующей системой

## Комментарии
Важно обеспечить точный учет токенов и стоимости для всех провайдеров с возможностью мониторинга через CLI.
