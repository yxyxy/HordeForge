# Подзадача P17.1: Интеграция Tree-sitter парсеров

## Описание
Необходимо интегрировать Tree-sitter парсеры в RAG-компонент для точного извлечения структурных элементов кода (функций, классов, методов).

## Цели
1. Добавить зависимости для Tree-sitter
2. Создать модуль для парсинга файлов с использованием Tree-sitter
3. Реализовать извлечение символов с помощью Tree-sitter запросов
4. Обновить `SymbolExtractor` для использования Tree-sitter

## Технические требования

### 1. Зависимости
- Добавить tree-sitter и языковые парсеры в зависимости проекта
- Поддерживать основные языки: Python, JavaScript, TypeScript, Java, Go, Rust, C++, C#

### 2. Модуль парсинга
- Создать отдельный модуль `rag/tree_sitter_parser.py`
- Реализовать универсальный интерфейс для парсинга различных языков
- Обеспечить кэширование парсеров для повышения производительности

### 3. Извлечение символов
- Использовать Tree-sitter запросы для точного извлечения символов
- Извлекать функции, классы, методы с их местоположением в файле
- Сохранять полную информацию о символах (имя, тип, местоположение, сигнатура)

### 4. Обновление SymbolExtractor
- Заменить текущую реализацию на основе регулярных выражений
- Обеспечить обратную совместимость с существующим интерфейсом
- Поддерживать текущий формат возвращаемых данных

## Подробное описание

### 1. Зависимости
Необходимо добавить следующие зависимости в `requirements.txt`:
```
tree-sitter==0.21.1
tree-sitter-python==0.23.6
tree-sitter-javascript==0.23.1
tree-sitter-typescript==0.23.3
tree-sitter-java==0.21.0
tree-sitter-go==0.23.2
tree-sitter-rust==0.23.2
tree-sitter-cpp==0.23.0
tree-sitter-c-sharp==0.23.0
```

### 2. Модуль парсинга
Создать `rag/tree_sitter_parser.py` с классом `TreeSitterParser`, который:
- Автоматически определяет язык файла по расширению
- Загружает соответствующий парсер
- Выполняет парсинг и возвращает AST
- Кэширует парсеры для повторного использования

### 3. Извлечение символов
Создать `rag/symbol_extractor_tree_sitter.py` с классом `TreeSitterSymbolExtractor`, который:
- Использует Tree-sitter запросы для извлечения символов
- Поддерживает различные типы символов (определения и ссылки)
- Возвращает структурированные данные о символах
- Использует запросы из Local RAG как основу

### 4. Обновление SymbolExtractor
Обновить `rag/symbol_extractor.py` чтобы:
- Использовать TreeSitterSymbolExtractor как основной движок
- Сохранить текущий интерфейс для обратной совместимости
- Добавить опциональный фоллбэк к старой реализации
