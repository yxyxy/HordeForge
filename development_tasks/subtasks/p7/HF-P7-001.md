# HF-P7-001 — API Rate Limiting

- Priority: P0
- Estimate: 2d
- Dependencies: P6 Complete
- Status: **DONE** (2026-03-10)

## Objective

Реализовать rate limiting middleware для защиты API от злоупотреблений и DDoS атак.

## Scope

- Rate limiting на уровне Gateway (FastAPI middleware)
- Поддержка различных лимитов для разных endpoints
- Tenant-based rate limiting
- Configuration-driven limits
- Graceful degradation

## Out of Scope

- OAuth/OIDC (отдельная задача HF-P7-002)
- Kubernetes-level rate limiting (можно добавить позже)

## Подзадачи

- [x] Реализовать rate limiter backend:
  - [x] In-memory rate limiter для dev/small deployments
  - [x] Redis-based rate limiter для production
  - [x] Abstraction layer для backend switching
- [x] Добавить middleware для FastAPI:
  - [x] Request counting
  - [x] Window-based (sliding window) algorithm
  - [x] Token bucket algorithm option
- [x] Конфигурирование лимитов:
  - [x] Per-endpoint limits
  - [x] Per-tenant limits
  - [x] Per-IP limits
  - [x] Environment-based defaults
- [x] Обработка exceeded limits:
  - [x] Return 429 status code
  - [x] Include Retry-After header
  - [x] Include rate limit info in headers
- [x] Покрыть unit tests:
  - [x] Test rate limiting logic
  - [x] Test middleware integration
  - [x] Test configuration loading
  - [x] Test Redis backend

## Критерии приемки

1. Rate limiting применяется ко всем public endpoints
2. 429 возвращается при превышении лимита
3. Tenant isolation работает корректно
4. Redis backend работает в production
5. Unit tests покрывают key functions
