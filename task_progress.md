# Задача: Запуск сканирования GitHub issue и пайплайна обработки

## Анализ системы

После анализа кода и документации установлено:

1. В системе есть `issue_scanner` задача, которая запускается каждые 15 минут (900 секунд)
2. Задача связана с `backlog_analysis_pipeline` пайплайном
3. Система уже инициализирована (проведена инициализация, RAG загружен)
4. Гетвей запущен на порту 800

## Команда для запуска

Для запуска процесса сканирования текущих открытых issue и запуска пайплайна их решения нужно выполнить:

```bash
curl -X POST http://localhost:8000/cron/run-due \
  -H "Content-Type: application/json" \
  -H "X-Operator-Key: local-operator-key" \
  -H "X-Operator-Role: operator" \
  -H "X-Command-Source: api"
```

Эта команда запустит все запланированные задачи, которые должны были выполниться, включая `issue_scanner`, который в свою очередь получит открытые issue из GitHub (используя настройки из инициализации) и запустит `backlog_analysis_pipeline` для каждого подходящего issue.
